Этот модуль представляет собой узел ROS2, который обрабатывает облака точек и строит для них ограничивающие боксы, публикуя результат в виде сообщений для дальнейшего использования или визуализации. Вкратце, программа реализует следующее:
Входные данные:

    Облака точек: Узел подписывается на топик object_point_cloud, который публикует сообщения типа ObjectPointClouds. Эти сообщения содержат облака точек объектов, их классы, идентификаторы слежения, и вероятности.

    Параметры камеры: Через параметр frame_id задаётся система координат камеры для визуализации боксов.

Логика работы:

    Подписка на облака точек: Узел подписывается на топик с сообщениями ObjectPointClouds. Когда приходит новое сообщение, запускается функция listener_callback, которая выполняет следующие шаги:

    Преобразование облаков точек:
        Преобразование облаков точек из формата ROS (PointCloud2) в формат Open3D для удобной обработки.
        Применяется алгоритм DBSCAN для удаления шумов из облаков точек. Алгоритм использует плотностный кластерный анализ, фильтруя изолированные точки как шум.

    Построение ограничивающего бокса:
        После очистки облака точек строится минимальный ориентированный ограничивающий бокс с помощью Open3D.
        Далее происходит коррекция размеров бокса, если размеры известны для данного объекта (например, для объектов с заданными tracking_id).

    Публикация сообщений:
        Результаты публикуются в два топика:
            В топик seg_track публикуется сообщение типа SegTrack, содержащее ограничивающие боксы объектов с их позициями, ориентациями и размерами.
            В топик bounding_box_markers публикуется сообщение типа MarkerArray, которое визуализирует боксы и текстовые метки в RViz.

Выходные сообщения:

    Сообщения SegTrack: Включают информацию о позициях и размерах объектов:
        Позиция и ориентация бокса в виде geometry_msgs/Pose.
        Размеры бокса: ширина, высота, глубина.
        Идентификатор класса и идентификатор слежения объекта.
    Сообщения MarkerArray: Используются для визуализации боксов в RViz, отображая их как кубы (Marker.CUBE) с текстовыми метками, указывающими ID объекта.
